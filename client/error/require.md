# PathClash 태블릿 인게임 레이아웃 수정 요청

현재 태블릿 대응을 위해 중앙 고정 세로 viewport 방식으로 레이아웃을 수정했는데, 문제가 하나 남아 있습니다.

## 현재 상태

- 로비 화면은 태블릿에서도 정상적으로 보입니다.
- 데스크탑에서도 정상입니다.
- 모바일에서도 정상입니다.
- 하지만 **태블릿 실기기에서 인게임 화면만 UI가 비정상적으로 커집니다.**
- 그 결과:
  - 상단 HUD / 하단 스킬 UI가 지나치게 커지고
  - 실제 보드가 차지하는 공간이 상대적으로 작아 보이며
  - 전체적으로 인게임만 비율이 깨져 보입니다.

## 중요한 관찰

- **데스크탑 크롬 개발자도구의 태블릿 에뮬레이션에서는 정상처럼 보입니다.**
- **실제 태블릿 크롬에서는 인게임만 커집니다.**
- 즉, 문제는 단순한 width breakpoint 문제가 아니라,
  **실기기 브라우저 viewport 계산(vh/dvh/svh/safe-area/address bar) 차이** 또는
  **인게임 레이아웃이 여전히 실제 화면 기준(vw/vh)으로 계산되는 문제**로 보입니다.

---

# 수정 목표

## 핵심 목표

로비와 인게임 모두 **동일한 중앙 고정 세로 viewport 기준**으로 렌더링되게 수정해주세요.

즉:

- 모바일: 세로 전체 사용
- 태블릿: 가운데 고정된 세로 게임 화면
- 데스크탑: 현재처럼 중앙 정렬
- **특히 인게임도 로비와 동일한 viewport 규칙을 반드시 따라야 함**

---

# 요구사항

## 1. 인게임도 로비와 동일한 최상위 viewport 컨테이너를 사용

현재 로비는 정상인데 인게임만 커지는 것으로 보아,
인게임 레이아웃이 별도 기준으로 렌더링되는 것 같습니다.

원하는 구조:

- `outer shell`
  - 전체 화면
  - 검은 배경
  - 가운데 정렬
- `game viewport`
  - width: 100%
  - max-width: 480px ~ 540px
  - height: 100%
  - 세로형 고정
  - overflow hidden

**인게임도 반드시 이 동일한 `game viewport` 안에서만 렌더링되게 바꿔주세요.**

---

## 2. 인게임 내부에서 viewport 단위(vw/vh/dvh/svh) 사용 제거 또는 최소화

특히 인게임 UI에서 아래를 점검해주세요:

- 보드 크기
- 상단 HUD
- 하단 스킬 패널
- HP 영역
- 버튼 높이
- 폰트 크기
- padding / margin / gap

현재 이 중 일부가 아마 `vw`, `vh`, `dvh`, `svh`, `calc(100vh - ...)` 같은 방식으로 계산되고 있는 것 같습니다.

### 원하는 방식

인게임 내부 요소는 **화면 기준이 아니라 부모 viewport 기준**으로 계산되게 바꿔주세요.

예:

- `%`
- `max-width`
- `aspect-ratio`
- `clamp()`
- flex/grid
- 부모 기준 height 분배

즉, **실제 브라우저 화면 높이 기준으로 커지지 않도록** 바꿔주세요.

---

## 3. 보드는 항상 부모 viewport 안에서 적정 최대 크기만 차지하도록 제한

태블릿에서 상하 HUD가 커지면서 보드가 작아 보이는 문제가 있습니다.

### 원하는 방식

- 보드는 viewport 안에서 남는 공간을 사용하되
- `max-width`, `max-height`, `aspect-ratio: 1 / 1`로 제한
- 태블릿에서 절대 과도하게 줄어들지 않도록
- HUD나 스킬 패널 때문에 보드가 너무 작아지지 않도록 비율 조정

즉, **인게임 전체에서 보드가 가장 중요한 요소로 보이게** 해주세요.

---

## 4. 인게임 HUD / 스킬 UI는 태블릿에서 자동 확대되지 않게 수정

현재 태블릿 실기기에서 아래 UI가 비정상적으로 큽니다.

- 상단 진행바 / LOBBY 버튼
- ATTACK / RUN 박스
- HP 표시
- 하단 스킬 카드
- 마나 바
- 텍스트

### 원하는 방식

태블릿에서는 로비처럼 “같은 세로 모바일 화면을 큰 기기 가운데에 띄운 느낌”이 나야 합니다.

즉:

- UI가 태블릿이라고 더 커지면 안 됨
- 로비와 인게임의 체감 스케일이 같아야 함
- 인게임 UI만 따로 확대되지 않게 해주세요

---

## 5. 실기기 브라우저의 동적 viewport / safe-area 문제 고려

데스크탑 개발자도구 에뮬레이션과 실기기 크롬이 다르게 보이는 문제를 해결해야 합니다.

가능하면 다음도 점검해주세요:

- `100vh` 대신 더 안전한 구조 사용
- 필요 시 `height: 100%` 기반 컨테이너 체인 구성
- safe-area inset 영향 점검
- 주소창/탭바가 있는 안드로이드 태블릿 크롬 환경에서도 안정적으로 보이게 수정

단, 핵심은 CSS 해킹보다
**인게임 전체가 부모 viewport 기준으로만 렌더링되도록 만드는 것**입니다.
