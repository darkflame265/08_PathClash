현재 능력대전 점수 시스템을 클래시로얄 스타일의 “아레나 진행도 기반 점수 손실 시스템”으로 변경하고 싶습니다.

기존처럼 아레나별로 승리/패배 점수를 고정하지 말고,
같은 아레나 안에서도 현재 점수 위치에 따라 패배 점수가 점진적으로 커지도록 수정해주세요.

의도:

- 아레나 진입 직후에는 점수 하락 부담을 낮춰 성장감을 제공
- 다음 아레나에 가까워질수록 긴장감을 증가
- 유저가 아레나에 일정 시간 머무르도록 유도
- 초반 인플레이션 완화
- 가짜 AI 상대 연승으로 점수가 너무 빨리 오르는 현상 방지

규칙:

1. 승리 점수

- 승리 시 획득 점수는 고정값 사용

모든 아레나는 승리 시 30점 얻는 거 고정.

2. 패배 점수

- 패배 시 잃는 점수는 현재 아레나 진행도(percent)에 따라 증가
- 아레나 시작 지점에서는 매우 적게 잃고,
  다음 아레나 직전에서는 많이 잃도록 구현

예시:
진행도 0% → -5
진행도 25% → -10
진행도 50% → -18
진행도 75% → -27
진행도 95% → -40

- 진행도 기준으로 선형 보간(linear interpolation) 사용
- 현재 점수가 아레나 범위 내 어디쯤인지 계산해서 패배 점수 결정

예시:
아레나4 범위가 900~1399이면:
currentProgress =
(currentScore - arenaMin) / (arenaMax - arenaMin)

3. 추가 조건

- 점수는 음수로 내려가지 않도록 처리
- 아레나 최소 점수 아래로는 떨어지지 않도록 보호

4. 밸런스 조정이 쉽도록

- 하드코딩 최소화
- 아레나 데이터/점수 테이블을 상수 배열로 분리
- 추후 승패 점수 수정이 쉽게 구조화

현재 코드 스타일 유지하면서 안전하게 리팩토링해주세요.
