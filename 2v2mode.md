# 2v2 모드

## 1. 개요

2v2 모드는 기존 대결전의 규칙과 흐름을 그대로 유지하되, 플레이어 수가 2명 대 2명으로 확장된 모드이다.

- 레드팀 2명
- 블루팀 2명
- 총 플레이어 수 4명

기본적인 승부 방식, 경로 작성 방식, 이동 및 충돌 규칙은 기존 대결전과 동일하다.  
즉, 새로운 규칙을 많이 추가하는 모드가 아니라, 기존 대결전 구조를 4인 팀전으로 확장한 형태로 이해하면 된다.

---

## 2. 기본 규칙

### 2-1. 팀 구성

- 레드팀 2명, 블루팀 2명으로 나뉜다.
- 각 팀 내부에는 기존 대결전과 동일하게 역할이 존재한다.
  - 공격자
  - 도망자

### 2-2. 기본 전장 규칙

- 맵 크기는 기존 대결전과 동일하게 5x5이다.
- 장애물 생성 규칙도 기존 대결전과 동일하다.
- 생성되는 장애물 개수는 4개이다.
- 장애물 생성 조건 및 배치 방식은 기존 대결전 알고리즘을 그대로 사용한다.

추가로, 기존 대결전에서 적용하던 시작 위치 안전 조건도 2v2에 맞게 확장해서 유지해야 한다.

- 기존 대결전에서는 빨강 시작 위치와 파랑 시작 위치 각각에 대해,
  즉시 1칸 이동 가능한 방향이 최소 2개 이상인지 검사했다.
- 2v2 모드에서는 플레이어가 4명이므로,
  레드1, 레드2, 블루1, 블루2의 시작 위치 각각에 대해
  즉시 1칸 이동 가능한 방향이 최소 2개 이상인지 검사해야 한다.

즉, 장애물 생성 조건은 팀 기준이 아니라 플레이어 개인 기준으로 검사해야 한다.

### 2-3. 시작 위치

기존 대결전에서는 각 팀 플레이어가 보드의 왼쪽 끝과 오른쪽 끝에서 시작했다.  
2v2 모드에서는 이 기존 시작 위치를 기준으로 같은 팀 두 명이 위아래로 한 칸씩 나뉘어 시작한다.

배치 원칙:

- 기존 대결전 시작 위치를 기준점으로 삼는다.
- 같은 팀 두 플레이어는 그 기준점에서
  - 한 명은 한 칸 위
  - 한 명은 한 칸 아래
  위치에서 시작한다.
- 즉, 같은 팀 두 플레이어가 세로로 한 칸 간격을 두고 붙어 있는 형태가 된다.

이 규칙에 따라 레드팀 두 명은 보드 한쪽 끝 라인에서 위아래로 나뉘어 시작하고,
블루팀 두 명도 반대편 끝 라인에서 같은 방식으로 위아래로 나뉘어 시작한다.

### 2-4. 경로 작성 및 이동

- 경로 작성 방식은 기존 대결전과 동일하다.
- 이동 및 충돌 판정도 기존 대결전과 동일하다.
- 단, 플레이어 수가 4명이므로 충돌 판정은 모든 플레이어를 기준으로 확장되어야 한다.

---

## 3. 시각 표시 규칙

### 3-1. 내 말

- 내 말은 기존 대결전과 마찬가지로 녹색 outline으로 표시된다.

### 3-2. 아군의 말

- 아군의 말은 소속 팀 색으로 표시된다.
- 레드팀 아군이면 빨강 outline
- 블루팀 아군이면 파랑 outline

### 3-3. 적군의 말

- 적군의 말도 해당 팀 색으로 표시된다.
- 레드팀 적이면 빨강 outline
- 블루팀 적이면 파랑 outline

즉, 2v2 모드에서는 말의 outline 색만 보고도 다음을 구분할 수 있어야 한다.

- 내 말인지
- 같은 팀인지
- 상대 팀인지
- 상대 팀이 어느 색 팀인지

---

## 4. 팀 경로 공유

같은 팀 플레이어끼리는 경로 작성 시간 동안 서로의 작성 중인 경로를 볼 수 있어야 한다.

- 이는 협동전에서 이미 사용 중인 아군 경로 공유 기믹을 참고하여 구현한다.
- 내 팀원은 실시간으로 어떤 경로를 작성 중인지 확인할 수 있어야 한다.
- 적 팀의 작성 중 경로는 보이지 않아야 한다.

즉, 경로 정보 공개 범위는 다음과 같다.

- 내 경로: 나에게 보임
- 아군 경로: 같은 팀에게 보임
- 적군 경로: 작성 중에는 보이지 않음

---

## 5. 게임 구조 원칙

2v2 모드는 기본적으로 기존 대결전의 확장판이어야 한다.

즉 다음 원칙을 유지해야 한다.

- 기존 대결전의 조작감과 템포를 유지할 것
- 맵 크기와 장애물 규칙을 크게 바꾸지 않을 것
- 새로운 복잡한 기믹을 추가하기보다, 팀 플레이 구조를 중심으로 확장할 것

이 모드는 새로운 장르의 모드가 아니라, 기존 대결전을 4인 팀전 형태로 발전시킨 모드로 설계한다.

---

## 6. UI 구성

2v2 모드는 기존 대결전 UI를 기반으로 하되, 팀원 정보까지 동시에 표시할 수 있도록 배치를 확장해야 한다.

### 6-1. 아군 UI

기존 대결전에서는 내 역할, 닉네임, HP를 표시하는 박스가 하나였다.  
2v2 모드에서는 같은 팀이 2명이므로, 아군 UI 박스 안에 팀원 정보까지 함께 표시해야 한다.

구성 원칙:

- 박스 좌측: 내 닉네임, 내 HP
- 박스 중앙: 팀 공통 역할 표시 박스
  - 공격자 / 도망자
- 박스 우측: 팀원의 닉네임, 팀원 HP

즉, 기존 1인용 상태 박스를 2인 팀용 상태 박스로 확장하는 구조이다.

### 6-2. 적군 UI

적군 UI도 동일한 원칙으로 구성해야 한다.

- 박스 좌측: 상대 한 명의 닉네임, HP
- 박스 중앙: 상대 팀 역할 표시 박스
- 박스 우측: 다른 상대 한 명의 닉네임, HP

즉, 아군 박스와 적군 박스는 같은 구조를 가져야 하며, 좌우에 두 플레이어 정보가 배치되고 중앙에는 역할 정보가 들어간다.

---

## 7. 구현 시 참고 사항

### 7-1. 기존 대결전 재사용 범위

재사용 가능한 것:

- 5x5 보드 구조
- 장애물 생성 로직
- 경로 작성 규칙
- 이동 처리
- 충돌 판정 기본 구조
- 역할 시스템
- 대결전 기본 UI 컴포넌트 스타일

추가로 확장해야 하는 것:

- 팀 단위 매칭 구조
- 4명 방 상태 관리
- 팀별 경로 실시간 공유
- 2인 팀 UI
- 4인 충돌 처리

### 7-2. 협동전에서 참고할 요소

협동전에서 이미 구현된 아래 요소는 참고 가치가 있다.

- 아군 경로 실시간 공유
- 팀 기반 상태 표현
- 다인 플레이 소켓 동기화 방식

단, 2v2 모드는 협동전처럼 공동 목표를 수행하는 구조가 아니라, 기존 대결전의 승부 구조를 유지하는 팀전이다.

---

## 8. 핵심 요약

- 2v2 모드는 기존 대결전의 4인 팀전 확장판이다.
- 레드팀 2명, 블루팀 2명으로 구성된다.
- 맵은 5x5, 장애물 4개로 기존 대결전과 동일하다.
- 각 플레이어 시작 위치마다 즉시 1칸 이동 가능한 방향이 최소 2개 이상이어야 한다.
- 시작 위치는 기존 대결전 시작 위치를 기준으로 같은 팀 두 명이 위아래 한 칸씩 나뉘어 배치된다.
- 경로 작성, 이동, 충돌 규칙은 기존 대결전을 최대한 유지한다.
- 같은 팀끼리는 경로 작성 중 서로의 경로를 볼 수 있다.
- 내 말은 녹색 outline으로 표시된다.
- 팀원과 적군은 팀 색 기준 outline으로 표시된다.
- UI는 좌우에 두 명의 닉네임/HP, 중앙에 팀 역할 박스를 두는 구조로 확장한다.
