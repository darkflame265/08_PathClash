내 웹 프로젝트의 인게임 화면을 반응형 레이아웃으로 수정해줘.

문제:

- 내 컴퓨터에서는 정상인데, 더 작은 노트북이나 태블릿에서는 게임 보드가 화면 밖으로 돌출됨
- 보드, HUD, 버튼, 채팅창, 오버레이가 현재 디스플레이 크기에 맞게 자동으로 줄어들거나 재배치되어야 함
- 피그마 오토 레이아웃처럼 화면 크기에 따라 자연스럽게 맞춰졌으면 함

수정 목표:

- 휴대폰, 태블릿, 작은 노트북, 데스크탑에서 모두 인게임이 화면 안에 들어오게 만들기
- 보드가 항상 우선적으로 화면 안에 보이도록 하기
- 가로폭/세로높이가 부족하면 컴포넌트 크기, 패딩, 폰트, 버튼 크기를 함께 축소하기
- 필요하면 레이아웃을 세로 스택 또는 2단 구조로 전환하기
- overflow 때문에 핵심 UI가 잘리거나 가려지지 않게 하기
- 단순히 zoom/transform scale로만 때우지 말고, 레이아웃 자체를 responsive하게 조정하기
- 16:9, 13인치 노트북, 태블릿 세로/가로, 모바일 세로 기준으로 확인해줘

작업 방식:

- 먼저 현재 인게임 관련 컴포넌트와 CSS 구조를 읽고 병목 원인을 분석해줘
- 그 다음 실제 코드 수정까지 진행해줘
- 특히 보드 크기를 viewport 또는 부모 컨테이너 기준으로 계산하도록 바꿔줘
- 고정 px 위주 스타일이 있다면 clamp(), min(), max(), grid/flex, aspect-ratio 등을 활용해 개선해줘
- 수정 후 어떤 파일을 왜 바꿨는지 설명해줘
