# In-Game i18n + Lobby Fix Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Lang toggle이 로비뿐 아니라 인게임 전체에 적용되고, 구글 계정 연동 관련 업그레이드 메시지도 언어에 맞게 번역되도록 수정한다.

**Tech Stack:** React, TypeScript, Zustand, Vite

### Task 2: useLang 훅을 store 기반으로 변경

**Files:**
- Modify: `client/src/hooks/useLang.ts`

**Context:**
`useLang`이 현재 `useState`로 독립적인 상태를 관리함. store의 `lang`/`setLang`을 사용하도록 변경.

**Step 1: useLang.ts 전체 교체**

```typescript
import { useGameStore } from '../store/gameStore';
import { translations } from '../i18n/translations';

export function useLang() {
  const lang = useGameStore((s) => s.lang);
  const setLang = useGameStore((s) => s.setLang);

  const toggleLang = () => {
    setLang(lang === 'en' ? 'kr' : 'en');
  };

  return { lang, toggleLang, t: translations[lang] };
}
```

### Task 6: LobbyScreen의 upgradeMessage를 reactive하게 번역

**Files:**
- Modify: `client/src/components/Lobby/LobbyScreen.tsx`

**Context:**
현재 `guestAuth.ts`의 `resolveUpgradeFlowAfterRedirect()`가 한국어 하드코딩 문자열을 `result.message`로 반환함.
LobbyScreen에서 이를 직접 state에 저장하기 때문에 언어 변경 후에도 원래 언어로 고정됨.

해결책: `upgradeMessage: string` state 대신 `upgradeResult: UpgradeResolution` state를 저장,
렌더 시점에 `t`를 통해 reactive하게 메시지를 생성.

**Step 1: UpgradeResolution 타입 import 추가**

파일 상단에:
```typescript
import type { UpgradeResolution } from '../../auth/guestAuth';
```

**Step 2: state 변경**

기존:
```typescript
const [upgradeMessage, setUpgradeMessage] = useState("");
```

변경:
```typescript
const [upgradeResult, setUpgradeResult] = useState<UpgradeResolution>({ kind: "none" });
```

**Step 3: upgradeMessage 파생 helper 추가**

컴포넌트 바깥 (또는 안쪽)에 helper 함수 추가:

```typescript
function getUpgradeDisplayMsg(result: UpgradeResolution, t: Translations): string {
  if (result.kind === "upgrade_ok") return t.upgradeOk;
  if (result.kind === "switch_ok") return t.switchOk(result.profile.wins, result.profile.losses);
  if (result.kind === "auth_error") return t.authError;
  return "";
}
```

컴포넌트 내 렌더 직전에:
```typescript
const upgradeMessage = getUpgradeDisplayMsg(upgradeResult, t);
```

**Step 4: useEffect 내 setUpgradeMessage 호출 교체**

기존의 `setUpgradeMessage(result.message)` / `setUpgradeNotice(result.message)` 호출들을 `setUpgradeResult(result)` 로 교체.

`upgradeNotice`도 동일하게 처리: `setUpgradeNotice` state는 `switch_ok` 여부만 bool로 저장해도 됨.
하지만 `UpgradeNoticeDialog`는 `message` prop을 받으므로, `upgradeMessage`를 재사용:

```typescript
// useEffect 안
if (result.kind === "upgrade_ok" || result.kind === "switch_ok") {
  applyProfileToStore(result.profile, setAuthState);
  setUpgradeResult(result);
  if (result.kind === "switch_ok") {
    setShowUpgradeNotice(true);  // bool로 변경
  }
  return;
}
setUpgradeResult(result);  // auth_error
```

`upgradeNotice` state를 `showUpgradeNotice: boolean`으로 변경:
```typescript
const [showUpgradeNotice, setShowUpgradeNotice] = useState(false);
```

`UpgradeNoticeDialog` 호출:
```typescript
{showUpgradeNotice && (
  <UpgradeNoticeDialog
    message={upgradeMessage}
    onClose={() => { setShowUpgradeNotice(false); setUpgradeResult({ kind: "none" }); }}
    t={t}
  />
)}
```

`handleLinkGoogle` / `handleLogout` 핸들러에서도 `setUpgradeMessage("")` → `setUpgradeResult({ kind: "none" })` 교체.
