# Patch Notes Process

이 문서는 PathClash 패치노트 갱신 규칙을 정리한 문서입니다.
로비 패치노트 UI와 변경점 작성 방식을 일관되게 유지하는 것이 목적입니다.

## 기본 원칙

- 패치노트는 로비 하단 버튼으로만 확인합니다.
- 새 버전이 있으면 버튼에 `NEW` 배지가 표시됩니다.
- 강제 팝업은 사용하지 않습니다.
- KR / EN 둘 다 관리합니다.
- 실제로 바뀐 사항만 적습니다.
- 읽는 사람이 바로 이해할 수 있게 간단히 씁니다.

## 갱신 시 같이 바꿔야 하는 것

패치노트를 갱신할 때는 아래를 함께 확인합니다.

1. 패치노트 버전 키
2. 우측 상단 날짜 표기
3. KR 문구
4. EN 문구
5. `NEW` 배지 동작 확인

관련 파일:
- [LobbyScreen.tsx](c:/08_PathClash/client/src/components/Lobby/LobbyScreen.tsx)
- [LobbyScreen.css](c:/08_PathClash/client/src/components/Lobby/LobbyScreen.css)

## 문구 작성 규칙

### 공통
- 길게 설명하지 말고 무엇이 바뀌었는지만 씁니다.
- 한 줄에 한 변경점만 씁니다.
- KR/EN은 의미를 맞추되, 직역보다 읽기 쉬운 쪽을 우선합니다.
- KR은 반드시 UTF-8 한글로 저장합니다.

### 마나 코스트 변경 표기
- 형식: `기존값 -> 변경값`
- 예:
  - `가드 마나 코스트: 4 -> 2`
  - `Guard mana cost: 4 -> 2`

### 버프 / 너프 표시 규칙
- 마나 코스트가 내려가면:
  - KR: `(버프)`
  - EN: `(Buff)`
  - 초록색 표시
- 마나 코스트가 올라가면:
  - KR: `(너프)`
  - EN: `(Nerf)`
  - 빨간색 표시

예:
- `가드 마나 코스트: 4 -> 2 (버프)`
- `양자 도약 마나 코스트: 3 -> 4 (너프)`

## 패치노트에 넣기 좋은 항목

- 신규 스킬 추가
- 스킬 코스트 조정
- 판정 버그 수정
- UI/모바일 조작 개선
- SFX / 시각 효과 추가
- 보상 정책 변경

## 패치노트에 넣지 않아도 되는 항목

- 내부 리팩터링만 한 경우
- 문서 추가만 한 경우
- 사용자 체감이 거의 없는 내부 정리

## 작성 예시

### KR
- `투명화 마나 코스트: 8 -> 4 (버프)`
- `용암지대 마나 코스트: 4 -> 6 (너프)`
- `능력대전 모바일 UI를 개선했습니다.`

### EN
- `Void Cloak mana cost: 8 -> 4 (Buff)`
- `Inferno Field mana cost: 4 -> 6 (Nerf)`
- `Improved Ability Battle mobile UI.`

## 갱신 체크리스트

1. 실제 코드 변경이 먼저 반영됐는지 확인
2. KR/EN 문구 작성
3. 날짜 갱신
4. 버전 키 증가
5. `NEW` 배지 동작 확인
6. 로비에서 패치노트 모달 열어서 UI 깨짐 없는지 확인
